# 데드락

## 로그 해석
- 2개의 워커 쓰레드가 자원 A, B에 대해서 하나씩 점유
- 이어서 각 워커 쓰레드가 점유하지 않은 자원 B, A를 각각 기다림
- 이것이 교착상태가 되며 두 쓰레드는 더 이상 진행하지 못하며 BLOCKED 상태가 되어버림

### [Worker-Thread-1]
- lock [Shared_Memory_A]
- LOCK ACQUIRED: [Shared_Memory_A]. (Holding...)
- processing critical data in Memory A...
- Need resource [Socket_Pool_B] to finish job.
- WAITING for [Socket_Pool_B]... (Status: BLOCKED)

### [Worker-Thread-2]
- lock [Socket_Pool_B]
- LOCK ACQUIRED: [Socket_Pool_B]. (Holding...)
- Establishing network connections in Pool B...
- Need resource [Shared_Memory_A] to write logs.
- WAITING for [Shared_Memory_A]... (Status: BLOCKED)
